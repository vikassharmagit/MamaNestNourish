Deployment Method
- The local project is packaged as a .tar.gz archive and copied to AWS.
- The archive is extracted into: /home/ubuntu/MamaNestNourish
- Docker builds the production image from Dockerfile:
  docker compose build
- Docker starts/restarts the app with:
  docker compose up -d
- The container restart policy is:
  restart: unless-stopped
- Docker service is enabled through systemd, so containers with restart policy can come back after instance reboot.

Docker Deployment Option
- The project includes Docker support.
- Dockerfile builds a production Node.js 20 image.
- docker-compose.yml runs the app on port 3000 and maps host port 3000 to container port 3000.
- Container runtime data is mounted at:
  ${MAMANEST_RUNTIME_DIR:-./runtime}:/app/runtime
- Inside the container, saved user/profile/plan data is written to:
  /app/runtime/user-records.json
- On the current AWS instance, MAMANEST_RUNTIME_DIR should be set to:
  /home/ubuntu/MamaNestNourish-runtime
- The same image can be moved to a new EC2 instance, Amazon ECR, ECS, or App Runner.
- On a new EC2 instance, the required setup becomes Docker, environment variables, persistent runtime storage, and nginx or an AWS load balancer.
